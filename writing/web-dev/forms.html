Improving on HTML Forms

When engaging visitors on a website, a lot of engagement is one way--the website conveying information to the visitor--but sometimes a two way engagement is necessary. Sometimes visitors need to contact persons affiliated with the website and sometimes persons affiliated with the website want or need to collect information from the visitors. These contacts and collections can take the shape of contact forms, job applications, idea submission forms, guest books and much more (Garofalo).

The most popular way to facilitate this kind of communication is through HTML Forms. According to w3.org, “an HTML form is a section of a document containing normal content, markup, special elements called controls (checkboxes, radio buttons, menus, etc.), and labels on those controls. Users generally "complete" a form by modifying its controls (entering text, selecting menu items, etc.), before submitting the form to an agent for processing (Forms).” Online forms, in general, are invaluable when needing to easily collect and organize data (Garofalo).

HTML forms, in particular, are becoming more powerful. The HTML5 Forms utilized today began as Web Forms 2.0, created by the Web Hypertext Applications Technology Working Group (WHATWG). The latest iteration, HTML5 Forms, added 13 new input types to that initial Web Forms 2.0. Those newer input types are: search, tel, url, email, datetime, date, month, week, time, datetime-local, number, range, and color. These new inputs don’t just lend more power to HTML Forms but also provide another layer of semantic richness and useability to visitors (Satrom).

There are many other ways to improve the user experience when creating HTML Forms. Ideas to contemplate when creating and improving forms include:

  • Know the audience and make form questions relevant to them
  • Give information with the form, including why the form should be filled and what is meant by certain queries
  • Make the form as simple as possible
  • Order the form logically
  • Show examples in form fields (one of HTML5 Forms’ inputs can do this)
  • Allow for full keyboard interactivity
  • Use clear error messages
  • Give feedback (highlight mandatory fields, color code as the form is being filled, etc.)

These are just some of the ways to make HTML Forms more powerful (Puri). There are many more. However, keep in mind: HTML Forms are especially useful when the visitors using them are considered.
